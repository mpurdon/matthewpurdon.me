# Project Notes

## Mockups And Layout Decisions

For homepage, layout, or major visual decisions, create a disposable mockup before editing production components.

- Put mockups in `mockups/`.
- Use a single HTML review harness with a visible toolbar.
- The toolbar should include previous/next navigation, version count, version title, and a short description of what changed.
- Keep each version visually comparable in the same viewport.
- Render screenshots beside the HTML, usually `mockups/<name>.png` and `mockups/<name>-full.png`.
- Only port the approved version into `src/` after review.
